The money math (reference)

  • Sign: status → approved, sets approved_at, fires Joby estimate_approved, emails OFFICE_EMAIL, pushes "Estimate Signed". (Web's sign is broken — wrong key.)
  • Send: status → sent; web then polls every 10s for the remote signature.
  • Convert: estimate → new invoice; web optionally merges existing job-invoice items (Keep & Add / Replace).
  • Collect deposit: inserts a completed payments row, sets deposit_collected, and moves the linked job to holding; returns remaining_balance. Correct keys here (amount_collected, payment_method).

⚠ Screen-level drift flags

  • Web estimate signature is broken — captureSignature sends signature_data; /estimates/:id/sign reads signature → 400. (Same wrong-key class as invoice/job.)
  • Web "Present GBB Options" is mis-wired — opens the broken signature modal instead of presenting tiers. Android has a real PresentTiersScreen + POST /estimates/:id/select-tier; web reads /tiers for display but never selects.
  • Web "Attach Photo" is broken — posts a multipart file to an endpoint that wants JSON {url} and has no multer → 400. Android sends {url} → works.
  • collect-deposit here is the CORRECT path (amount_collected/payment_method), unlike the Job-Detail deposit (wrong keys + non-existent estimate_id).
  • deposit-settings is Android-only on the detail screen; web configures deposits in the builder. Android also guards edit-after-signed with a re-sign warning.
  • UNVERIFIED: the builder save payload (POST /estimates / PUT /estimates/:id), GBB tier authoring, select-tier body beyond {tier_id}.
