⚠ Screen-level drift flags

  • Poll cadence differs: web reloads jobs+techs together every 30 s; Android polls techs every 15 s and loads jobs once (on open / resume / manual refresh). Tech positions are fresher on Android.
  • Tech-pin navigation is Android-only. Web tech pins are display-only (name + current job / "Available"); Android tech pins open the tech's in_progress job via onJob(current_job_id).
  • Two different hardcoded Google Maps API keys in client source — web JS SDK key (…kJiNQ, LiveMap.jsx:5) and Android geocoding key (…vQjyk, LiveMapScreen.kt:118). Both committed in plaintext; rotate/restrict is a security follow-up.
  • GET /gps/live is a bare array filtered to techs that pinged within 5 min (role technician, is_active), LATERAL-joined to their current in_progress job. Depends on the field app sending POST /gps/ping — UNVERIFIED here.
  • Silent pin drops: jobs with neither coords nor a geocodable address, and failed geocodes, are dropped with no user-visible message on both surfaces.
